Ruby

Desenvolvida no Japão em 1995, por Yukihiro “Matz” Matsumoto com o intuito de ser uma linguagem de script para a web mais poderosa do que o Perl e mais orientada a objetos do que o python (na época). Sua sintaxe foi inspirada principalmente por: Python, Perl, Smalltalk, Eiffel, Ada e Lisp. A utilização da linguagem ruby em web é para desenvolvimento back-end. Ruby é uma linguagem open-source.

PHP

O PHP como é conhecido hoje, é na verdade o sucessor para um produto chamado PHP/FI. Criado em 1994 por Ras­mus Lerdof, a primeira encarnação do PHP foi um simples con­junto de binários Common Gateway Interface (CGI) escrito em linguagem de programação C. Originalmente usado para a­com­pa­nha­mento de visitas para seu currículo online, ele no­me­ou o conjunto de scripts de "Personal Home Page Tools" mais frequentemente referenciado como "PHP Tools."

Python

A história do Python começa com Guido Van Rossum a iniciar o seu desenvolvimento em 1989 e a começar a implementá-lo em fevereiro de 1991, altura em que foi lançada a primeira versão pública: 0.9.0. A versão 1.0 foi lançada em janeiro de 1994, a versão 2.0 foi lançada em outubro de 2000 e a versão 3.0 foi lançada em dezembro de 2008.

Java

A linguagem Java surgiu a partir de uma pesquisa financiada pela Sun Microsystems em 1991, iniciada sob o codinome de projeto Green. Java é uma linguagem Open Source e não possui um único dono, todas novas atualizações da linguagem são feitas através da JCP (Java Community Process), formada por um conjunto de empresas.

Como eu acho que vai ser a programação daqui a 20 anos

Lendo um pouco sobre a história das Linguagens de Pro­gra­mação, vi que desde seu início na década de 50 com o FORTRAN, LISP e COBOL , ela foi sofrendo alterações notáveis com o sur­gi­mento de novas linguagens, que por sua vez foram se aperfeiçoando até chegar nos dias atuais.

A programação é um trabalho que em grupo facilita as coisas, com um grupo formado as pessoas passam a desenvolver suas soft e hard skills, além de aprender com os outros é possível aprender com os próprios erros. Além do mais a programação vem se tornando mais "fácil" com o passar dos anos, por isso foi possível adquirir conhe­cimento em vários locais diferentes, com vários especialistas diferentes.

Com o passar dos anos, acredito que as linguagens de programação se tornarão mais fáceis de serem "consumidas" e atribuídas para conhecimento próprio. Porém um ponto que não pode escapar é que a programação se torna fácil para quem busca conhecimento "além dos horizontes", dado pelo fato de que a programação está em uma constante evolução.
